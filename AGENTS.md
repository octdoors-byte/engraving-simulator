# Repo working rules for Codex

## 必須ルール（最優先）
- **ファイル削除・移動・リネーム・大きな整理**は、必ず事前にユーザーへ確認してから実行する（`git rm` / `apply_patch` の Delete / Move を含む）。
- **変更を入れたら、原則その都度コミット**する（小さく、目的が分かる単位で）。
- コミットは関係するファイルだけを `git add` し、無関係な差分（ログ・一時ファイル・行末変換など）を混ぜない。

## 運用メモ
- `logs/` 等のローカル生成物はコミットしない（必要なら `.gitignore` を提案する）。
- 迷ったら「いま何をコミットするか」「削除して良いか」を先に確認する。
