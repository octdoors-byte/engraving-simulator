# 開発サーバー起動確認手順

## 問題の原因と対策

### よくある原因

1. **ポート5174が既に使用されている**
   - 他のプロセスがポートを使用している可能性
   - 解決方法: 既存のプロセスを終了するか、別のポートを使用

2. **Node.js/npmがインストールされていない**
   - Node.js 20以上23未満が必要
   - 解決方法: Node.jsをインストール

3. **パスの問題（日本語文字を含むパス）**
   - PowerShellの文字エンコーディングの問題
   - 解決方法: スクリプトを改善済み

4. **依存関係がインストールされていない**
   - `node_modules`が存在しない
   - 解決方法: `npm install`を実行

## 起動方法

### 方法1: 改善されたスクリプトを使用（推奨）

```powershell
.\start-local.ps1
```

### 方法2: 直接コマンドで起動

```powershell
cd "プロジェクトのパス"
npm run dev
```

### 方法3: バッチファイルを使用

```cmd
start-local.bat
```

## トラブルシューティング

### ポートが使用されている場合

```powershell
# ポート5174を使用しているプロセスを確認
netstat -ano | findstr :5174

# プロセスを終了（PIDを確認してから）
taskkill /PID <PID番号> /F
```

### ログを確認

ログファイル: `資料\ログ\dev-server.log`

```powershell
Get-Content "資料\ログ\dev-server.log" -Tail 50
```

### 依存関係を再インストール

```powershell
npm install
```

### Node.jsのバージョンを確認

```powershell
node --version
npm --version
```

必要なバージョン: Node.js 20以上23未満

## 正常に起動した場合

以下のメッセージが表示されます：

```
VITE v5.x.x  ready in xxx ms

➜  Local:   http://localhost:5174/
➜  Network: http://xxx.xxx.xxx.xxx:5174/
```

ブラウザで `http://localhost:5174` にアクセスできます。
