# ビルドエラー確認：手動実行手順

## 現在の状況

ビルドが失敗していますが、エラーメッセージが表示されていません。
「✓ 258 modules transformed.」と表示されているので、モジュールの変換は成功していますが、
その後の処理（ファイル出力など）で失敗している可能性があります。

---

## エラー詳細を確認する方法

### 方法1: コマンドプロンプトから直接実行

1. **コマンドプロンプトを開く**
   - Windowsキー + R → `cmd` と入力 → Enter

2. **プロジェクトフォルダに移動**
   ```cmd
   cd /d "C:\Users\owner\OneDrive\ドキュメント\ラクリプ専用フォルダ\【NEW】名入れ刻印シミュレーター"
   ```

3. **.env.production ファイルを作成**
   ```cmd
   echo VITE_BASE_PATH=/simulator/ > .env.production
   ```

4. **ビルドを実行（エラーを確認）**
   ```cmd
   npm run build
   ```

5. **エラーメッセージを確認**
   - 表示されたエラーメッセージをすべてコピーしてください

---

### 方法2: ビルドログをファイルに出力

1. **コマンドプロンプトを開く**

2. **プロジェクトフォルダに移動**
   ```cmd
   cd /d "C:\Users\owner\OneDrive\ドキュメント\ラクリプ専用フォルダ\【NEW】名入れ刻印シミュレーター"
   ```

3. **ビルドログをファイルに出力**
   ```cmd
   echo VITE_BASE_PATH=/simulator/ > .env.production
   npm run build > build.log 2>&1
   type build.log
   ```

4. **build.log ファイルを確認**
   - エラーメッセージが記録されています

---

## よくある原因と解決方法

### 1. ファイルパスの問題

**症状:** 長いパス名や日本語を含むパスでエラーが発生

**解決方法:**
- プロジェクトを短いパスに移動（例: `C:\projects\engraving-simulator`）

### 2. ディスク容量不足

**症状:** ビルド中にディスク容量が不足

**解決方法:**
- ディスクの空き容量を確認
- 不要なファイルを削除

### 3. ファイル権限の問題

**症状:** `dist/` フォルダへの書き込み権限がない

**解決方法:**
- `dist/` フォルダを削除してから再ビルド
- 管理者権限でコマンドプロンプトを開く

### 4. TypeScript のエラー

**症状:** ソースコードにエラーがある

**解決方法:**
- エラーメッセージを確認して修正

### 5. 依存関係の問題

**症状:** `node_modules` が正しくインストールされていない

**解決方法:**
```cmd
npm install
```

## 次のステップ

1. **エラーメッセージを確認**
   - 上記の方法1または方法2でエラーメッセージを確認

2. **エラーメッセージを共有**
   - エラーメッセージの内容を教えていただければ、具体的な解決方法を提案します

3. **一時的な回避策**
   - `dist/` フォルダを削除してから再ビルド
   - 管理者権限でコマンドプロンプトを開いて実行

---

まずは、**方法1（コマンドプロンプトから直接実行）** を試して、エラーメッセージを確認してください。
