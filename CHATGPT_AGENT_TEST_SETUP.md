# ChatGPT Agent への動作テスト依頼

このドキュメントは、ChatGPT Agentにデザインシミュレーターアプリケーションの動作テストを実施してもらうための指示書です。

## 依頼内容

GitHub Pagesにデプロイされているデザインシミュレーターアプリケーションの包括的な動作テストを実施してください。

### テスト対象

- **URL**: https://octdoors-byte.github.io/engraving-simulator/
- **プロジェクト名**: 名入れ刻印シミュレーター
- **技術スタック**: Vite + React (SPA)
- **デプロイ環境**: GitHub Pages

### テストの目的

1. デプロイされたアプリケーションが正常に動作していることを確認
2. 最近実装された機能（管理画面ドロップダウン、アコーディオン、セクション別保存など）が正しく動作することを確認
3. SPAルーティングが正常に機能することを確認
4. 潜在的な問題やバグを発見

## テスト実施手順

### 1. 基本準備

1. ブラウザを開く（推奨: Chrome、Firefox、Edge）
2. 開発者ツール（F12）を開く
3. コンソールタブとネットワークタブを確認できるようにする

### 2. テスト項目

詳細なテスト項目は `CHATGPT_AGENT_TEST_PROMPT.txt` を参照してください。

主なテストカテゴリ：
- 基本アクセステスト
- ナビゲーションテスト
- 管理画面機能テスト
- SPAルーティングテスト
- レスポンシブテスト
- エラーハンドリングテスト

### 3. テスト結果の記録

各テスト項目について、以下を記録してください：
- ✅ 正常に動作
- ❌ 問題が発見された
- ⚠️ 確認が必要

問題が発見された場合は、以下を記録：
- 問題の詳細説明
- 期待される動作
- 実際の動作
- スクリーンショット（可能であれば）
- コンソールエラー（あれば）
- ネットワークエラー（あれば）

## 最近実装された機能（重点的にテスト）

### 1. ナビゲーション改善
- 「デザイン発行履歴」が一番左に配置され、目立つスタイルになっている
- 管理画面が右上のアイコンボタンに統合され、ドロップダウンメニューでアクセス可能

### 2. 基本設定ページの改善
- カテゴリ設定とヘッダー/フッター設定がアコーディオン形式（初期状態で閉じている）
- 各セクションに個別の「保存する」ボタンが追加
- ページ最下部に「すべて保存」ボタンが配置
- 共通説明（お客様向け）に例データが設定されている

### 3. ロゴサイズ変更機能
- 基本設定でロゴサイズを変更できる
- 変更後、ヘッダーのロゴサイズが即座に反映される

## 期待される動作

### ナビゲーション
- すべてのナビゲーションリンクが正常に動作する
- 管理画面ドロップダウンメニューが開閉する
- アクティブなページが視覚的に強調表示される

### 基本設定ページ
- アコーディオンが正常に開閉する
- セクション別の保存ボタンが機能する
- 「すべて保存」ボタンが機能する
- 例データが表示される

### SPAルーティング
- 直接URLアクセスでページが表示される
- ブラウザの戻る/進むボタンが正常に動作する
- 404エラーが発生しない

## トラブルシューティング

### ページが表示されない場合
1. GitHub Pagesのデプロイが完了しているか確認
2. ブラウザのキャッシュをクリア
3. シークレットモードでアクセス

### コンソールエラーが表示される場合
1. エラーメッセージを記録
2. エラーが発生する操作を記録
3. ネットワークタブでリソースの読み込み状況を確認

### SPAルーティングが動作しない場合
1. 404.htmlが正しく生成されているか確認
2. 直接URLアクセスでテスト
3. ブラウザの戻る/進むボタンでテスト
