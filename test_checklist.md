# テストチェックリスト（簡単版）

目的: テンプレート登録からPDF発行まで、実際に問題なく動くかを確認する。

---

## 1. 事前準備
- [ ] `/admin/templates` に入れる
- [ ] `template.json` と背景画像を用意
- [ ] 背景画像のファイル名が `template.background.fileName` と一致
- [ ] 背景画像のサイズが `background.canvasWidthPx / canvasHeightPx` と一致

## 2. テンプレート登録
- [ ] `template.json` と背景画像を同時にドラッグ&ドロップで登録できる
- [ ] エラーが出ずに登録完了メッセージが出る
- [ ] 一覧に「表示名 / テンプレキー / 状態 / 更新日」が表示される

## 3. テンプレート管理
- [ ] 状態（下書き / テスト済み / 公開中）を切り替えられる
- [ ] テスト / スマホ表示 / PC表示が開く
- [ ] モノクロ設定が反映される

## 4. シミュレーター（お客様画面）
- [ ] 背景が表示される
- [ ] ロゴがアップロードできる（PNG/JPEG/WEBP）
- [ ] 5MB超は拒否される
- [ ] トリミングを開いて調整できる
- [ ] 色クリックで背景透過ができる
- [ ] 透過なしに戻せる
- [ ] ロゴをドラッグで動かせる
- [ ] ロゴのサイズ変更ができる
- [ ] 枠外だと発行できない
- [ ] 枠より大きい場合、発行時にエラーが出る

## 5. PDF発行（確認用）
- [ ] 発行ボタンでPDFが自動ダウンロードされる
- [ ] 見た目がシミュレーターと一致する
- [ ] 位置/サイズ(mm)が表示される
- [ ] デザインIDが最下部に表示される

## 6. デザイン発行履歴
- [ ] 一覧に履歴が表示される
- [ ] PDFプレビューが開く
- [ ] プレビュー後にダウンロードできる

---

## Codexで確認できる項目（コード確認）
- [ ] テンプレJSONのバリデーション
- [ ] PDFの座標計算
- [ ] PDFのデザインID / 位置サイズの表示処理
- [ ] 発行時エラー（枠より大きい場合）

## 手動でしか確認できない項目
- [ ] 実際の見た目（画面 / PDF）
- [ ] ドラッグ操作のしやすさ
- [ ] 画像の透過結果
